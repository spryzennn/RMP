# Теоретические вопросы по мобильной разработке

## 1) Что такое мобильное приложение и какие основные типы мобильных приложений существуют?

**Ответ:**  
Мобильное приложение — это программа, разработанная для работы на смартфонах, планшетах и других портативных устройствах.

Основные типы:
- **Нативные** — создаются под конкретную операционную систему.
- **Кроссплатформенные** — один код работает на нескольких платформах.
- **Веб-приложения** — адаптированные сайты, которые работают в браузере.
- **Гибридные** — сочетают элементы веб- и нативной разработки.

**Подвопрос №1**  
**Чем нативное приложение отличается от кроссплатформенного?**  
Нативное приложение пишется под одну конкретную платформу и лучше использует её возможности. Кроссплатформенное приложение создаётся сразу для нескольких платформ на общей кодовой базе.

**Подвопрос №2**  
**Почему веб-приложения относятся к мобильным?**  
Потому что они адаптируются под экран телефона и запускаются через браузер без обязательной установки из магазина приложений.

**Подвопрос №3**  
**В чём преимущество гибридных приложений?**  
Они позволяют ускорить разработку и частично переиспользовать код для разных платформ.

**Подвопрос №4**  
**Какой тип приложения обычно выбирают для максимальной производительности?**  
Обычно выбирают нативное приложение, потому что оно лучше использует ресурсы устройства и системные API.

## 2) Какие платформы для разработки мобильных приложений существуют?

**Ответ:**  
Основные платформы — это **Android** и **iOS**. Также используются кроссплатформенные фреймворки, например **Flutter**, **React Native** и **Xamarin**.

**Подвопрос №1**  
**На каких языках обычно пишут Android-приложения?**  
Чаще всего используют Kotlin и Java.

**Подвопрос №2**  
**На каких языках обычно пишут iOS-приложения?**  
Обычно используют Swift и Objective-C.

**Подвопрос №3**  
**Зачем нужны кроссплатформенные фреймворки?**  
Чтобы писать одно приложение сразу для Android и iOS с меньшими затратами времени и ресурсов.

**Подвопрос №4**  
**Какую среду разработки используют для Android?**  
Обычно Android Studio.

## 3) Какие особенности обладают мобильные приложения?

**Ответ:**  
Мобильные приложения работают в условиях ограниченных ресурсов, используют сенсорный интерфейс, часто обращаются к камере, GPS и датчикам, а также должны учитывать нестабильный интернет и жизненный цикл приложения.

**Подвопрос №1**  
**Почему нужно экономить батарею в мобильном приложении?**  
Потому что мобильные устройства работают от аккумулятора, и лишние вычисления быстро уменьшают время автономной работы.

**Подвопрос №2**  
**Почему важно учитывать нестабильный интернет?**  
Потому что сеть может пропадать или работать медленно, и приложение должно корректно обрабатывать такие ситуации.

**Подвопрос №3**  
**Что значит сенсорный интерфейс?**  
Это интерфейс, где управление происходит через касания, жесты и свайпы.

**Подвопрос №4**  
**Что такое жизненный цикл приложения?**  
Это последовательность состояний, через которые проходит приложение: создание, запуск, работа, пауза, остановка и уничтожение.

## 4) Перечислите и опишите основные компоненты Android-приложения

**Ответ:**  
Основные компоненты Android-приложения — это **Activity**, **Service**, **Broadcast Receiver** и **Content Provider**.

- **Activity** — экран приложения с интерфейсом.
- **Service** — фоновая работа без интерфейса.
- **Broadcast Receiver** — реакция на системные события.
- **Content Provider** — доступ и управление общими данными.

**Подвопрос №1**  
**Какой компонент отвечает за экран с интерфейсом?**  
Activity.

**Подвопрос №2**  
**Для чего нужен Service?**  
Для выполнения длительной фоновой работы без пользовательского интерфейса.

**Подвопрос №3**  
**Что делает Broadcast Receiver?**  
Он получает и обрабатывает широковещательные сообщения и системные события.

**Подвопрос №4**  
**Зачем нужен Content Provider?**  
Для безопасного обмена данными между приложениями или внутри системы.

## 5) Что такое Activity в Android? Какой метод вызывается при создании Activity?

**Ответ:**  
Activity — это компонент Android, который представляет один экран с пользовательским интерфейсом. При создании Activity вызывается метод `onCreate()`.

**Подвопрос №1**  
**Что обычно делают в `onCreate()`?**  
Инициализируют интерфейс, подключают элементы UI и подготавливают объекты для работы экрана.

**Подвопрос №2**  
**Сколько раз вызывается `onCreate()`?**  
Обычно один раз при создании Activity, если она не пересоздаётся системой.

**Подвопрос №3**  
**Что идёт после `onCreate()`?**  
После него обычно вызывается `onStart()`.

**Подвопрос №4**  
**Можно ли Activity существовать без интерфейса?**  
Обычно Activity связана с интерфейсом, потому что она предназначена для отображения экрана пользователю.

## 6) Опишите жизненный цикл активности (Activity Lifecycle). Какой метод вызывается, когда активность становится видимой пользователю?

**Ответ:**  
Жизненный цикл Activity включает `onCreate()`, `onStart()`, `onResume()`, затем `onPause()`, `onStop()` и `onDestroy()`.

Когда активность становится видимой пользователю, вызывается `onStart()`. Когда она начинает взаимодействовать с пользователем — `onResume()`.

**Подвопрос №1**  
**Когда вызывается `onResume()`?**  
Когда Activity выходит на передний план и пользователь может с ней взаимодействовать.

**Подвопрос №2**  
**Когда вызывается `onPause()`?**  
Когда Activity частично теряет фокус и пользователь уходит на другой экран или приложение сворачивается.

**Подвопрос №3**  
**Когда вызывается `onStop()`?**  
Когда Activity становится невидимой пользователю.

**Подвопрос №4**  
**Зачем нужен жизненный цикл?**  
Чтобы приложение правильно управляло ресурсами, сохраняло состояние и не блокировало интерфейс.

## 7) Что такое REST API и как оно используется в мобильных приложениях?

**Ответ:**  
REST API — это способ обмена данными между приложением и сервером через HTTP-запросы, такие как GET, POST, PUT и DELETE. В мобильных приложениях REST API используется для получения и отправки данных, например новостей, профиля пользователя или погоды.

**Подвопрос №1**  
**Что такое JSON в REST API?**  
Это популярный формат передачи данных между клиентом и сервером.

**Подвопрос №2**  
**Почему REST API удобно использовать в мобильных приложениях?**  
Потому что оно позволяет отделить клиентскую часть от серверной и легко обновлять данные.

**Подвопрос №3**  
**Какие ошибки может вернуть API?**  
Например, ошибки авторизации, отсутствия ресурса или проблемы сервера.

**Подвопрос №4**  
**REST API работает только в мобильных приложениях?**  
Нет, его используют также в веб-приложениях, десктопных программах и на сервере.

## 8) Объясните концепцию MVC (Model-View-Controller) в контексте мобильной разработки

**Ответ:**  
MVC — это архитектурный подход, где:
- **Model** отвечает за данные и бизнес-логику.
- **View** отвечает за отображение интерфейса.
- **Controller** обрабатывает действия пользователя и связывает Model и View.

**Подвопрос №1**  
**Что относится к Model?**  
Данные, бизнес-правила и работа с источниками данных.

**Подвопрос №2**  
**Что относится к View?**  
Элементы интерфейса: кнопки, поля ввода, экраны, разметка.

**Подвопрос №3**  
**Что делает Controller?**  
Он принимает действия пользователя, обрабатывает их и обновляет данные или интерфейс.

**Подвопрос №4**  
**Зачем нужна такая архитектура?**  
Чтобы код был более понятным, модульным и удобным для тестирования.

## 9) Что такое XML?

**Ответ:**  
XML — это язык разметки, который используется для описания структурированных данных. В Android он применяется для макетов интерфейса, строк, стилей и конфигурации приложения.

**Подвопрос №1**  
**Почему XML часто используют в Android?**  
Потому что он удобно описывает структуру интерфейса и другие настройки приложения.

**Подвопрос №2**  
**Что такое layout-файл?**  
Это XML-файл, в котором описывается внешний вид экрана приложения.

**Подвопрос №3**  
**Можно ли хранить в XML данные?**  
Да, XML подходит для хранения структурированных данных и конфигураций.

**Подвопрос №4**  
**Чем XML отличается от JSON?**  
XML более многословный и часто используется для разметки, а JSON — чаще для обмена данными через API.

## 10) Что такое корутины в Android-приложениях?

**Ответ:**  
Корутины — это механизм асинхронного программирования, который позволяет выполнять длительные операции, не блокируя главный поток. В Android они особенно полезны для сетевых запросов и работы с базой данных.

**Подвопрос №1**  
**Зачем корутины нужны в Android?**  
Чтобы приложение не зависало во время долгих операций и оставалось отзывчивым.

**Подвопрос №2**  
**Что такое главный поток?**  
Это поток, в котором обновляется пользовательский интерфейс. Его нельзя надолго блокировать.

**Подвопрос №3**  
**Почему корутины лучше колбэков?**  
Потому что код становится более читаемым и похожим на обычную последовательную логику.

**Подвопрос №4**  
**Где часто применяются корутины?**  
В сетевых запросах, загрузке данных и обращении к локальной базе данных.

## 11) Перечислите основные свойства виджетов UI мобильных приложений. Опишите их

**Ответ:**  
К основным свойствам относятся `layout_width`, `layout_height`, `text`, `visibility`, `onClick`, `gravity` и `layout_gravity`.

- `layout_width` и `layout_height` — ширина и высота элемента.
- `text` — текст на виджете.
- `visibility` — видимость элемента.
- `onClick` — обработчик нажатия.
- `gravity` и `layout_gravity` — выравнивание содержимого и элемента.

**Подвопрос №1**  
**Что означает `wrap_content`?**  
Элемент занимает столько места, сколько нужно его содержимому.

**Подвопрос №2**  
**Что означает `match_parent`?**  
Элемент занимает всё доступное пространство родителя.

**Подвопрос №3**  
**Чем отличается `gravity` от `layout_gravity`?**  
`gravity` выравнивает содержимое внутри элемента, а `layout_gravity` — сам элемент внутри родителя.

**Подвопрос №4**  
**Зачем нужен `visibility`?**  
Чтобы управлять отображением элемента: показать, скрыть или убрать его из разметки.

## 12) В чём разница между SQLite и Firebase Realtime?

**Ответ:**  
**SQLite** — локальная реляционная база данных, которая хранится на устройстве и может работать без интернета.  
**Firebase Realtime Database** — облачная NoSQL база данных, которая синхронизирует данные между клиентами в реальном времени и требует подключения к сети.

**Подвопрос №1**  
**Когда лучше использовать SQLite?**  
Когда данные должны храниться локально и быть доступны офлайн.

**Подвопрос №2**  
**Когда лучше использовать Firebase Realtime?**  
Когда нужно синхронизировать данные между пользователями или устройствами почти мгновенно.

**Подвопрос №3**  
**SQLite является SQL или NoSQL?**  
SQLite — это SQL-реляционная база данных.

**Подвопрос №4**  
**Firebase Realtime подходит для офлайн-режима?**  
Основной сценарий у неё облачный, поэтому она в первую очередь рассчитана на работу с интернетом.

## 13) Перечислите основные варианты хранения данных в мобильных приложениях. Опишите их

**Ответ:**  
Основные варианты хранения данных:
- **SharedPreferences** — простые пары «ключ-значение».
- **Internal Storage** — приватные файлы приложения.
- **External Storage** — внешние файлы, доступные пользователю.
- **Room / SQLite** — реляционные данные.
- **Firebase / REST API** — удалённое хранение на сервере.

**Подвопрос №1**  
**Для чего нужны SharedPreferences?**  
Для хранения простых настроек, например темы оформления или токена.

**Подвопрос №2**  
**Чем отличается внутреннее хранилище от внешнего?**  
Внутреннее хранилище приватно для приложения, а внешнее обычно используется для файлов, доступных пользователю или другим приложениям при наличии разрешений.

**Подвопрос №3**  
**Когда используют Room или SQLite?**  
Когда нужно хранить структурированные данные и выполнять запросы к ним.

**Подвопрос №4**  
**Что такое сетевое хранилище?**  
Это хранение данных на удалённом сервере, например через Firebase или собственное API.

## 14) Какие бывают уведомления в мобильных приложениях и как они работают?

**Ответ:**  
Основные виды уведомлений — **Toast**, **Snackbar** и **push-уведомления**.

- **Toast** — краткое всплывающее сообщение.
- **Snackbar** — сообщение с возможной кнопкой действия.
- **Push-уведомления** — системные уведомления, которые приходят даже при закрытом приложении.

**Подвопрос №1**  
**Чем Toast отличается от Snackbar?**  
Toast — это краткое сообщение без действий, а Snackbar может содержать кнопку и привязывается к интерфейсу.

**Подвопрос №2**  
**Что такое push-уведомления?**  
Это сообщения, которые система показывает пользователю через уведомления Android.

**Подвопрос №3**  
**Как работает push-уведомление?**  
Обычно сервер отправляет сообщение через сервис доставки, например Firebase Cloud Messaging.

**Подвопрос №4**  
**Почему push-уведомления полезны?**  
Потому что они позволяют информировать пользователя даже когда приложение не открыто.

**Ответ:**  
`AndroidManifest.xml` — это обязательный конфигурационный файл приложения. В нём описываются компоненты приложения, разрешения, версия SDK, а также основные параметры вроде имени, иконки и версии.

**Подвопрос №1**  
**Почему без манифеста приложение не работает?**  
Потому что система Android использует его для понимания структуры приложения и его прав.

**Подвопрос №2**  
**Что такое разрешения в манифесте?**  
Это доступы к возможностям устройства, например камере, сети или контактам.

**Подвопрос №3**  
**Какие компоненты обычно указываются в манифесте?**  
Activity, Service, Broadcast Receiver и Content Provider.

**Подвопрос №4**  
**Что такое minSdk и targetSdk?**  
Это минимальная и целевая версии Android, для которых приложение рассчитано.

## 16) СОП. Обработка событий

**Ответ:**  
СОП — это событийно-ориентированное программирование. В нём выполнение программы зависит от событий, например нажатий кнопок, касаний экрана или системных сообщений. В Android это реализуется через слушатели, такие как `setOnClickListener`.

**Подвопрос №1**  
**Что такое событие?**  
Это действие или изменение состояния, на которое программа должна отреагировать.

**Подвопрос №2**  
**Что такое listener?**  
Это объект или обработчик, который ждёт событие и выполняет нужный код.

**Подвопрос №3**  
**Почему этот подход удобен для Android?**  
Потому что интерфейс мобильных приложений построен вокруг действий пользователя.

**Подвопрос №4**  
**Пример события в Android?**  
Нажатие на кнопку, ввод текста или изменение состояния экрана.

## 17) Что такое Intent-filter?

**Ответ:**  
`intent-filter` — это элемент в `AndroidManifest.xml`, который указывает, на какие типы `Implicit Intent` может реагировать компонент. Он описывает действия, категории и данные, с которыми компонент умеет работать.

**Подвопрос №1**  
**Где указывается `intent-filter`?**  
В файле `AndroidManifest.xml` внутри описания Activity, Service или другого компонента.

**Подвопрос №2**  
**Какие элементы входят в `intent-filter`?**  
Обычно `action`, `category` и `data`.

**Подвопрос №3**  
**Зачем нужен `intent-filter` для Activity?**  
Чтобы система могла предложить это приложение для обработки подходящего действия, например открытия ссылки.

**Подвопрос №4**  
**`intent-filter` нужен только для запуска приложения?**  
Нет, он используется для обработки разных типов внешних и системных вызовов.

## 18) Что такое Intent? В чем разница между явным и неявным Intent?

**Ответ:**  
`Intent` — это механизм Android для запуска компонентов и передачи данных.

- **Явный Intent** указывает конкретный компонент.
- **Неявный Intent** описывает только действие и данные, а система сама подбирает подходящее приложение или компонент.

**Подвопрос №1**  
**Для чего используют явный Intent?**  
Для запуска конкретной Activity или Service внутри приложения.

**Подвопрос №2**  
**Для чего используют неявный Intent?**  
Для вызова внешнего приложения или системного действия, например открытия браузера.

**Подвопрос №3**  
**Можно ли передавать данные через Intent?**  
Да, данные передаются через extras.

**Подвопрос №4**  
**Чем Intent отличается от обычного вызова функции?**  
Intent работает как сообщение между компонентами Android, а не как обычный метод внутри класса.

## 19) Как работает механизм авторизации и аутентификации в мобильных приложениях?

**Ответ:**  
Аутентификация проверяет личность пользователя, а авторизация определяет его права доступа. После успешного входа сервер обычно выдаёт токен, например JWT, который приложение хранит и отправляет в заголовке `Authorization` при последующих запросах.

**Подвопрос №1**  
**Чем аутентификация отличается от авторизации?**  
Аутентификация отвечает на вопрос «кто ты?», а авторизация — «что тебе разрешено?».

**Подвопрос №2**  
**Что такое токен?**  
Это строка, которая подтверждает успешный вход пользователя и используется для дальнейших запросов.

**Подвопрос №3**  
**Зачем хранить токен на устройстве?**  
Чтобы не вводить логин и пароль при каждом запросе к серверу.

**Подвопрос №4**  
**Почему токен отправляют в заголовке `Authorization`?**  
Потому что сервер использует его для идентификации пользователя и проверки доступа к ресурсам.
